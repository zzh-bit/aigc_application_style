# Parallel Self 2.0 项目策划文档（开发执行版）

## 1. 项目定位与目标

- **项目名称**：Parallel Self 2.0（PS2）
- **项目定位**：基于 AIGC 的个人决策辅助与成长复盘系统，融合理性推演、情绪感知、记忆检索与洞察报告。
- **交付目标**：完成可用 Web 应用，并将完整静态产物打包为 Android APK，实现离线可用、联网增强。

核心愿景：在用户面对复杂选择时，提供“多角色辩论 + 记忆上下文 + 可视化推演 + 决策复盘”的完整闭环，持续提升决策质量与自我认知。

## 2. 产品原则与设计理念

1. **理性与情绪双轨并行**：决策质量由分析能力和情绪状态共同决定。
2. **内在冲突外化**：将“激进/保守/未来/导师”转化为可交互角色，降低思考盲区。
3. **本地优先与可持续可用**：核心数据本地存储，网络不可用时仍保留主流程能力。
4. **闭环而非单次问答**：从提问、推演到归档、洞察，形成长期成长系统。

## 4. 功能范围与模块映射（按 function.md 对齐）

### 4.1 多维决策支持

- 决策议会：多角色观点生成、主持人总结、会话归档
- 导师智库：导师人设库、独立对话线程、跨议题建议补充

### 4.2 个性化数据融合

- 记忆锚点：用户历史信息存储、检索、引用
- 情绪感知：文本规则版识别，触发呼吸引导等轻干预

### 4.3 时空对话系统

- 未来信件：写信、保存、触发、送达记录

### 4.4 可视化决策引擎

- 叙事推演：接受/拒绝/延迟等路径结果展示
- 决策树探索：时间线与树状分支联动浏览

### 4.5 核心系统能力

- 统一视觉风格（深色玻璃态）
- 成长洞察报告（主题、情绪、偏见提示）
- 深度设置（角色参数、通知与隐私策略）

## 5. 技术架构与打包策略

### 5.1 架构分层

- **Web 前端**：Next.js App Router，负责界面与状态流转
- **API/BFF**：`app/api/*`（开发态或部署态提供动态能力）
- **本地存储**：IndexedDB/localStorage（离线数据留存）
- **Android 壳**：WebView + WebViewAssetLoader 加载静态站点

### 5.2 关键约束（必须遵守）

1. `output: "export"` 导出的静态站点不包含 Node 运行时 API，APK 内不能直接运行 `app/api/*`。
2. APK 使用的 `out/` 必须由 `npm run build:android` 生成，确保资源路径为相对路径（`./_next/...`）。
3. AI 密钥只放服务端环境变量，前端和 APK 不可硬编码。

## 6. 分阶段实施计划（建议）

### 阶段 A：可演示闭环（优先）

目标：先跑通“提问 -> 议会 -> 推演 -> 归档 -> 洞察”的主链路。

- 完善议会与推演结果联动
- 将决策归档与洞察页面串联
- 验收标准：核心链路稳定可重复演示

### 阶段 B：能力补齐

- 新增/完善 API：`projection`、`insights`、`mentor`、`letters/trigger`、`letters/delivery`
- 完善 `lib/rag/memory-retriever.ts`、`lib/insights/metrics.ts`
- 验收标准：模块边界清晰，输入输出契约稳定

### 阶段 C：Android 打包联调

- `npm run build:android` 生成 `out/`
- 同步 `out/` 到 `android/ps2-shell/app/src/main/assets/web/`
- Gradle 组包并真机安装
- 验收标准：断网可打开主界面；联网可调用后端核心 API

### 阶段 D：发布前稳定性

- 回归主导航与关键交互
- 校验资源路径、接口容错、弱网表现
- 输出演示脚本与交付清单

## 8. 风险与应对

- **风险 1：静态导出资源路径错误导致样式/点击失效**  
  应对：强制使用 `npm run build:android`，并保留导出校验脚本。

- **风险 2：将 API 误认为可随 APK 离线运行**  
  应对：明确 API 外置部署，使用 `NEXT_PUBLIC_API_BASE_URL` 指向可达服务。

- **风险 3：功能面过大导致交付延迟**  
  应对：优先闭环链路，增量补齐高级能力，不阻塞打包主线。

- **风险 4：本地数据兼容与迁移问题**  
  应对：在 `lib/storage.ts` 设计版本字段与迁移逻辑。

## 9. 成功标准（项目完成定义）

当满足以下条件，视为本阶段项目成功：

1. Web 端完成核心闭环并可稳定演示；
2. APK 可安装、可打开、可离线浏览主流程；
3. 联网时可访问外部 BFF 并返回有效 AI 结果；
4. 文档、脚本、构建流程可复现，团队成员可独立执行。
