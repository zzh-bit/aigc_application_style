# Parallel Self 2.0 功能与亮点说明

## 1. 软件定位

Parallel Self 2.0 是一款面向“复杂决策场景”的多角色 AI 对话与复盘应用。  
它将“实时对话、决策推演、记忆管理、未来信件、数据洞察”整合为一个闭环，帮助用户从情绪与理性两条线同时推进决策质量。

---

## 2. 核心功能

### 2.1 多派系议会对话
- 用户输入议题后，激进派、保守派、未来派、主持人（可选导师）给出不同立场建议。
- 支持在线模型回复与本地兜底回复，保证弱网/离线情况下仍可继续使用。
- 可邀请不同导师风格参与同一议题，补充跨学科视角。

### 2.2 情绪监测与深呼吸干预
- 对用户输入进行情绪识别并驱动情绪盘动态变化。
- 当焦虑达到阈值时触发呼吸引导页面，帮助用户降压并恢复专注。
- 情绪状态支持会话级持久化，跨页面返回后保持连续。

### 2.3 决策推演（路径树）
- 围绕当前议题生成多条决策路径，并展示收益、风险、情绪预测。
- 显示各派系对每条路径的支持度与意见，辅助比较与取舍。
- 支持在线推演与本地回退，兼顾质量与稳定性。

### 2.4 记忆库（本地持久化）
- 用户可记录个人记忆条目（生活/工作/旅行/学习/情绪等）。
- 支持摘要与关键词，用于后续对话中的“引用记忆”增强。
- 采用本地存储机制，保障移动端场景下的数据可持续可回读。

### 2.5 未来信件
- 支持写给未来/回信过去，并设置触发条件（时间、情绪、事件）。
- 信件数据本地保存，支持在后续会话中持续使用。

### 2.6 数据洞察
- 基于“决策完成”后的归档记录，统计累计归档、周复盘频率、情绪分布与主题分布。
- 主题分布聚焦固定五类：生活、工作、旅行、学习、情绪。
- 支持 AI 解释与偏见卡片，辅助识别冲动决策、拖延规避、风险偏好等模式。

---

## 3. 关键亮点

### 3.1 决策闭环完整
从“讨论 -> 推演 -> 决策完成 -> 归档 -> 洞察 -> 下轮优化”形成可持续迭代链路，而非一次性问答。

### 3.2 情绪与理性并行
不仅给出方案建议，还关注用户情绪波动与干预时机，降低高压下误判概率。

### 3.3 多端与弱网友好
Web + Android WebView 一体化方案，支持本地回退逻辑，提升移动端可用性。

### 3.4 本地优先的数据策略
核心会话、记忆、信件、洞察基线数据可在本地持久化，减少对单一在线链路的依赖。

### 3.5 可扩展的 AI 接入架构
关键模块（议会对话、推演、洞察解释、偏见卡片）均支持模型增强，便于后续升级。

---

## 4. 典型使用流程

1. 在议会页输入现实议题并查看多派系建议。  
2. 打开推演页比较不同路径的收益与风险。  
3. 结合记忆引用与导师建议确定执行方案。  
4. 点击“决策完成”归档本轮对话。  
5. 进入数据洞察查看主题变化、情绪走势与偏见提醒。  
6. 基于洞察开启下一轮更高质量决策。

---

## 5. 适用场景

- 职业选择与城市去留  
- 学习规划与留学决策  
- 关系与生活平衡议题  
- 高压力时期的情绪化决策管理  
- 个人长期成长复盘与策略迭代
